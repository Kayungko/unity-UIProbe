# SCOPE-001 最小修改

## 规则

1. when 要新建文件/模块/抽象 → then 先搜同类并入已存在 // warn 孤儿文件 = 持续维护负担；不新建已有流程能承担的实体
2. when 要加字段/参数/钩子点 → then 指名至少一个真实读取方 // warn 无消费者 = schema 膨胀
3. when 见到两处相似代码 → then 并排写，不抽 // warn 2 处是预测，3 处才是证据
4. when 想为"以后"留占位/分支/选项 → then 删占位 // warn 未来需求 ≠ 当前需求
5. when 改 bug 顺手想清理/重命名 → then 只留与 bug 直接相关的 diff // warn 混合 diff 阻碍 review 与回滚
6. when skill/helper 想封装单次 MCP 调用 → then 直接调 MCP // warn 无固定工作流的封装 = 伪 DRY

原因：新实体是持续维护负担；Agent 写代码成本低，读懂 + 维护成本线性增长。

## 三问门 · 质疑清单（新增任何实体前一律过）

1. **删这一步** → 有验收失败？否 → 删
2. **并入已有文件** → 装得下？能 → 放
3. **真实调用方** → 现在存在？否 → 等
