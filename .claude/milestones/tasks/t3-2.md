## 描述

在 Unity Editor 内启 HttpListener(仅 127.0.0.1),暴露 /health、/rpc、/tools/list、/tools/describe,经 Dispatcher 把 /rpc 请求投递主线程调用 ToolRegistry。v0.1 仅 HTTP + loopback,WebSocket 留 v0.2。

1. UIProbe/Editor/Infrastructure/Bridge/HealthStatus.cs
   - status(ok)、serverId(guid,进程内新建绝不持久化)、pid、projectPath、uiProbeVersion、contractVersion、isCompiling、isUpdating、isPlaying。

2. UIProbe/Editor/Infrastructure/Bridge/SessionToken.cs
   - Bridge 启动生成一次性 token 写仅当前用户可读文件;/rpc 校验 header 防同机恶意进程直连。

3. UIProbe/Editor/Infrastructure/Bridge/UIProbeBridge.cs
   - HttpListener 绑 127.0.0.1 随机端口;后台线程收请求。
   - GET /health -> HealthStatus。
   - POST /rpc(校验 token)-> 解析 ToolRequest -> Dispatcher.Enqueue 调 ToolRegistry.Invoke -> ToolResult;错误码 UNITY_BUSY/MAIN_THREAD_TIMEOUT/DOMAIN_RELOAD_INTERRUPTED。
   - GET /tools/list、/tools/describe -> 代理 ToolRegistry。
   - 把端口/projectPath 写入发现文件供 Orchestrator 按 projectPath 路由。

4. UIProbe/Tests/Editor/BridgeEndpointTests.cs
   - 本地起 Bridge,curl 风格请求验证 /health 字段、/rpc token 校验、/tools 列表。

5. 不能动的边界
   - 仅监听 loopback;无 token 的 /rpc 拒绝;只读工具经此打通端到端。

## 实现偏差说明

- **跨模块 [Serializable] 修复（用户批准）**: /rpc 经 JsonUtility 序列化 ToolResult 时,嵌套自定义类字段(Error/Issues/Changes/Risks/ProgressLog 及 ToolSchema)若类型未标 `[System.Serializable]` 会被静默丢弃,导致结构化错误码无法回传(`Rpc_Times_Out_When_Not_Pumped` 即因 `result.Error` 为 null 而失败)。经用户确认采用"完整修复全部线上 DTO",给 ToolError/Issue/Change/Risk/LogEntry/ToolSchema 加 `[Serializable]`(纯增量,不改行为),一次修好 /rpc 全量错误/Issue/Change 回传,M4 Node 端直接受益。属 tool-contract 模块,已扩展本 task 写入路径并以 `--accept-check` 记录。
- **asmdef 引用扩展**: UIProbe.Editor 增加对 UIProbe.Core.Services 的引用,使 Bridge 能调用 ToolRegistry。
- **HealthStatus 字段用 camelCase**: JsonUtility 按字段名原样序列化,刻意对齐 wiki/Node 端消费契约(camelCase)。
- **SessionToken ACL**: 依赖用户级目录默认 ACL 实现"仅当前用户可读",更细粒度 ACL 收紧留后续。
- **ReserveLoopbackPort**: HttpListener 不支持绑 :0 自动选端口,先用 TcpListener 绑 :0 拿空闲端口再释放。
- **/tools/list 手工拼 JSON**: ToolDescriptor 未标 [Serializable],嵌进 wrapper 会序列化成空对象,故逐元素 ToJson 后手工拼数组(顶层 ToJson 可用,同 ToolRegistry.DescribeTool)。
- **UNITY_BUSY 未被覆盖**: v0.1 只读路径统一按 JobKind.Read 投递,无写工具触发 UNITY_BUSY,故端到端仅验证 MAIN_THREAD_TIMEOUT;UNITY_BUSY 路由留写工具接入时验证。
- **几何断言验收行**: 第 44 行为模板生成残留,与本 task 无关,忽略。
- **生产接线推迟 T3-3**: EditorApplication.update→Pump 接线与真实 projectPath/版本注入留 T3-3 生命周期任务。
