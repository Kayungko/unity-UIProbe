## 描述

用 Node/TypeScript 建立 MCP Server 工程骨架与到 Unity Bridge 的连接管理。Server 对外讲 MCP 协议,对内连 Bridge HTTP loopback,自身不碰 Unity API。

1. mcp-server/package.json + tsconfig.json
   - 依赖 MCP SDK(@modelcontextprotocol/sdk)、ws 暂不需(v0.1 HTTP)、测试框架(vitest 或 node:test)。
   - scripts:build/test/start。

2. mcp-server/src/bridge/BridgeConnection.ts
   - BridgeConnection(projectPath、baseUrl、sessionToken、lastServerId、state:Connecting/Ready/Reloading/Offline)。
   - 从发现文件读端口与 projectPath,从受控文件读 sessionToken(不写日志)。
   - 多实例按 projectPath 路由。

3. mcp-server/src/bridge/handshake.ts
   - Handshake():GET /health,校验 contractVersion major(不符直接拒绝并提示升级),记录 uiProbeVersion。

4. mcp-server/test/handshake.test.ts
   - 用 mock Bridge(本地 stub HTTP)验证:major 一致 -> ok;major 不符 -> 拒绝;Unity 未启动 -> UNITY_OFFLINE。

5. 不能动的边界
   - Server 不直接碰 Unity API;sessionToken 不外泄不写日志。
