## 描述

PrefabIndex 是后续多个能力的单一数据源(引用追踪/重复检测/嵌套总览/过滤扫描都基于它派生)。本任务从 UIProbeWindow_Indexer.cs 抽出查找 prefab、构建 folder tree、收集 Image/RawImage/Prefab/Material 引用、保存加载 IndexCache、搜索展开的非 UI 逻辑,落为 PrefabIndexService。务必先抽它以验证 ToolContract + Adapter 接缝 + 黄金样本闭环。

1. UIProbe/Core/Services/PrefabIndexData.cs（落点修正,见下方"实现偏差"）
   - PrefabIndex(Items、SchemaVersion、BuiltAt、RootPath);版本不符直接重建不迁移。folder 视图由 Items.FolderPath 派生,不另存 FolderTree(YAGNI)。
   - PrefabIndexItem(Guid、AssetPath、Name、FolderPath、ReferencedAssets:List<AssetRef>、ComponentSummary)。
   - PrefabIndexBuildOptions(RootFolders、Incremental);LoadCacheResult(Found/SchemaValid/Index)。

   > **实现偏差(build 阶段发现)**: 原计划数据类型落 `UIProbe/Data/PrefabIndexData.cs`,但 `UIProbe/Data/` 整体编译进 Editor-only 程序集(29 文件中 9 个引用 UnityEditor),`UIProbe.Core.Services`(全平台)无法引用其类型。故新数据类型改落 `UIProbe/Core/Services/PrefabIndexData.cs`(同文件名,目录修正使 Service 可见)。遗留 `UIProbe/Data/PrefabIndexData.cs` 的 Serializable* 类型保持原样不动。不另建第 6 个程序集(scope-discipline)。

2. UIProbe/Core/Services/PrefabIndexService.cs(经 IAssetGateway/IFileSystem 注入)
   - BuildIndex(options):构建索引,全量几千 prefab 走 LongRunning + jobId + 进度;支持增量。
   - LoadCache()/SaveCache(index):缓存读写,版本不符自动重建。
   - Search(index, query):按关键词搜索,把展开匹配的非 UI 逻辑从 Window 下沉到此。
   - GetPrefabDetail({guid|path}):返回单条 PrefabIndexItem,缺失回 TOOL_NOT_FOUND。
   - 仅由本 Service 持有 PrefabIndex,其他模块只读派生不另缓存。

3. UIProbeWindow_Indexer.cs 改造
   - 保留搜索框/收藏夹/树视图/批量选择按钮等 UI,业务数据改由 PrefabIndexService 提供。

4. UIProbe/Tests/Editor/PrefabIndexServiceTests.cs
   - 用 InMemoryAssetGateway 预置 prefab 资源表,验证 BuildIndex/Search/GetPrefabDetail。
   - 用黄金样本对一组预置 prefab 的索引输出做 diff 回归(行为零变化)。

5. 不能动的边界
   - 只读,不触发 Domain Reload,不修改 prefab。UI 交互行为须与迁移前一致(黄金样本保证)。

## 验收标准

- [ ] PrefabIndexService 提供 BuildIndex/LoadCache/SaveCache/Search/GetPrefabDetail
- [ ] PrefabIndex 含 SchemaVersion,版本不符自动重建不迁移
- [ ] 全量索引走 LongRunning+jobId+进度,支持增量
- [ ] Search 的非 UI 匹配逻辑从 Window 下沉到 Service
- [ ] GetPrefabDetail 缺失返回 TOOL_NOT_FOUND
- [ ] Service 经 Adapter 注入,可用内存假体单测
- [ ] UIProbeWindow_Indexer 的搜索框/收藏夹/树视图/批量选择 UI 保留且行为不变
- [ ] 黄金样本对索引输出 diff 全绿
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)

PrefabIndexService 抽离完成,经 Adapter 可单测,黄金样本回归证明 Indexer Tab 行为零变化,成为其他只读 Service 的底座。
