## 描述

当前项目无任何 asmdef,所有脚本进默认程序集,导致无法单元测试、无法清晰隔离 Editor 与运行时代码。本任务建立程序集边界,为黄金样本回归与 Adapter 单测奠基。这是 readiness 从 governance_ready 走向 execution_ready 的关键一步。

1. UIProbe/UIProbe.asmdef(核心运行时/可被编辑器引用的纯逻辑层)
   - 放 Core/Contract、Core/Services、Data 等不直接依赖 UnityEditor 的代码(凡能不依赖 UnityEditor 的尽量下沉到此)。
   - includePlatforms 不限定,allowUnsafeCode 按需。

2. UIProbe/Editor/UIProbe.Editor.asmdef
   - includePlatforms 仅 Editor。
   - 引用 UIProbe.asmdef。放 EditorWindow、UnityAdapters 的 Unity 实现、Bridge 等依赖 UnityEditor 的代码。

3. UIProbe/Tests/Editor/UIProbe.Tests.Editor.asmdef
   - includePlatforms 仅 Editor。
   - references 含 UIProbe.asmdef、UIProbe.Editor.asmdef、nunit.framework、UnityEngine.TestRunner、UnityEditor.TestRunner。
   - 引入 Unity Test Framework 包(确认 manifest.json 含 com.unity.test-framework)。

4. 现有脚本归位
   - 把现有 .cs 按依赖关系移动到对应 asmdef 目录下,确保编译通过。这是机械迁移,不改业务逻辑。先让现有 UIProbeWindow_*.cs 等留在 Editor 程序集,后续里程碑再逐步把纯逻辑下沉到运行时程序集。

5. 不能动的边界
   - 本任务只建程序集边界并让其编译通过,不重构业务逻辑,不抽 Service。
