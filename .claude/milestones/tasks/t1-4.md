## 描述

ToolRegistry 是工具体系的中枢:所有工具集中注册,MCP 与 UI 必须经它发现与调用,不得绕过。本任务建立 Registry 骨架并打通只读调用路径,为 M2 的只读 Service 接入做准备。

1. UIProbe/Core/Services/ToolRegistration.cs
   - 注册条目:ToolDescriptor + 工具实例工厂/单例引用。
   - 支持 Attribute 驱动发现(定义 [UIProbeTool] 特性),便于项目方扩展自有工具。

2. UIProbe/Core/Services/ToolRegistry.cs
   - Register(IUIProbeTool):登记并去重(同名冲突报错)。
   - ListTools(profile 可选过滤):返回 List<ToolDescriptor>。
   - DescribeTool(name):返回单个 ToolDescriptor,缺失回 TOOL_NOT_FOUND。
   - Invoke(ToolRequest):按 name 查找 -> Validate -> 按 Mode 走 Preview 或 Execute -> 返回 ToolResult。只读工具直接 Execute。
   - 通过构造注入 Adapter,转交给 ToolContext。

3. OperationTicket(为写操作两阶段预留)
   - Preview 阶段产出 ticket(含预期 Changes),Execute 凭 ticket 落地。v0.1 只读不强制,但骨架预留字段。

4. UIProbe/Tests/Editor/ToolRegistryTests.cs
   - 用内存假体 + 一个 fake 只读工具验证注册/发现/调用闭环。
   - 覆盖:注册后可 List/Describe;Invoke 不存在的工具回 TOOL_NOT_FOUND;Validate 失败回 Issues;只读 Invoke 返回 Success。

5. 不能动的边界
   - 本任务不接入真实业务 Service(留 M2),只用 fake 工具打通闭环。
