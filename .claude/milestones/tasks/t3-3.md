# T3-3: Domain Reload 恢复:serverId 信号 + job 中断 + capabilities 重报

> 由 milestone_gen 生成。

## 元数据

- **Agent**: core-unity-bridge
- **阶段**: exec
- **里程碑**: M3

## 描述

Domain Reload 是常态而非异常。本任务实现重载前后的优雅处理:beforeAssemblyReload 标记进行中 job 为 Interrupted、关 HttpListener、持久化必要状态;afterAssemblyReload 生成新 serverId、重建 Bridge、重报 capabilities。Orchestrator 通过 serverId 变化感知重载。

1. UIProbe/Editor/Infrastructure/Bridge/BridgeReloadHandler.cs
   - 订阅 AssemblyReloadEvents.beforeAssemblyReload:把进行中 job 标 Interrupted、关闭 HttpListener、持久化必要状态(不持久化 serverId)。
   - 订阅 afterAssemblyReload:生成新 serverId、重建 Bridge、重新注册工具并重报 capabilities。
   - 写操作遇 reload -> DOMAIN_RELOAD_INTERRUPTED 交人确认;ReloadSafe=true 的幂等只读可标记供 Orchestrator 自动重发。

2. 与 [InitializeOnLoad]
   - afterAssemblyReload 后自动重建 Bridge,保证重载后无需人工重启。

3. UIProbe/Tests/Editor/BridgeReloadTests.cs
   - 模拟 reload 生命周期:验证 serverId 变化、进行中 job 被标 Interrupted、重建后工具可再次发现。

4. 不能动的边界
   - serverId 每次 afterAssemblyReload 重新生成,绝不持久化;reload 期间端口短暂不可用属正常,由 Orchestrator 退避重试。

## 验收标准

- [ ] beforeAssemblyReload 标记进行中 job 为 Interrupted 并关闭 HttpListener
- [ ] afterAssemblyReload 生成新 serverId 并重建 Bridge、重报 capabilities
- [ ] serverId 绝不持久化,每次重载重新生成
- [ ] 写操作遇 reload 返回 DOMAIN_RELOAD_INTERRUPTED
- [ ] ReloadSafe 只读工具标记为可自动重发
- [ ] [InitializeOnLoad] 保证重载后 Bridge 自动重建无需人工重启
- [ ] BridgeReloadTests 覆盖 serverId 变化与 job 中断
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)

## 退出准则

Domain Reload 前后 Bridge 优雅中断与自动重建,serverId 变化可被外部感知,重载后工具可再次发现调用。

## 实现偏差说明

- **静态 Unity 依赖隔离到 BridgeBootstrap.cs**(同模块新增,unity-bridge):`[InitializeOnLoad]` + `AssemblyReloadEvents` 订阅 + `EditorApplication.update += Dispatcher.Pump` 集中此处,使 `BridgeReloadHandler` 逻辑保持可单测(直接调 OnBefore/OnAfter)。BridgeBootstrap 兑现 T3-2 推迟的生产 update→Pump 接线 + 真实 projectPath(`Path.GetDirectoryName(Application.dataPath)`)/版本注入。
- **Dispatcher 中断接缝落在 MainThreadDispatcher.cs**(同模块,unity-bridge,故仅声明写入路径无需跨模块批准):新增 `InterruptRunningJobs()` 取消并标 Interrupted 返回受影响 job;`DispatchJob` 增 `ReloadSafe`/`AutoResendable` 字段;`EnqueueLong` 增 `reloadSafe` 参数。
- **BridgeBootstrap batchmode 不自启**:`if (Application.isBatchMode) return;`——Bridge 仅服务交互式 Editor,避免测试/CI 期占端口与写发现文件副作用;测试直接驱动 handler/dispatcher,不经 bootstrap。
- **v0.1 薄注册表**:bootstrap 用 `new ToolRegistry(null,null,null)` 空注册保 /tools 可达,只读工具接线随 MCP MVP 补全。
- **OnBeforeAssemblyReload 把 Current 置 null**:Stop 后清引用,afterAssemblyReload 必重建新实例(新 serverId),不复用旧 Bridge。
- 验收末条 geometric assertions(Vector3/quaternion/mesh bounds)为 milestone 模板伪影,与本任务无关,忽略。
- Domain Reload 在 batchmode 测试内无法完整复现,故对 handler 回调逻辑做单元级验证(serverId 变化/job 分类中断/HttpListener 关闭/重载后工具可发现);手动 Editor 内实测留后续记录。
