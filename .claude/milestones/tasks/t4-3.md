## 元数据

- **Agent**: core-tool-registry

## 描述

把 M2 的三个只读 Service 包装成符合 ToolContract 的 UIProbeTool 并注册进 ToolRegistry,使其经 Bridge 与 MCP Server 端到端可被 AI 调用。这是 v0.1 的验收闭环:AI -> MCP -> Bridge -> Registry -> Service。

1. UIProbe/Core/Tools/(每个工具一个文件,[UIProbeTool] 标注)
   - build_prefab_index / get_prefab_index_status / search_prefabs / get_prefab_detail(包装 PrefabIndexService;build 为 LongRunning)。
   - find_asset_references(包装 AssetReferenceService)。
   - run_ui_checks / get_check_results / export_report(包装 UICheckService;export 写受控目录)。
   - 每个工具实现 DescribeParams(JSON Schema)/Validate/Preview(只读等于 Execute)/Execute,Descriptor 标 ReloadSafe=true、UndoCapability=None。

2. UIProbe/Tests/Editor/ReadOnlyToolsTests.cs
   - 经 ToolRegistry.Invoke 验证每个工具的 schema、validate、execute 行为(用内存假体 + 黄金样本)。

3. 端到端手动验收
   - 启 Unity + Bridge,启 MCP Server,用 MCP 客户端 ListTools 看到上述工具,CallTool 跑通 search_prefabs/find_asset_references/run_ui_checks 拿到结构化结果。
   - 记录 AI 向验收清单(每个工具一条 happy path + 一条错误路径)。

4. 不能动的边界
   - 全部只读;不引入写操作(留 v0.2)。
