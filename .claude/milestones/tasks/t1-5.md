# T1-5: 黄金样本回归基线机制

## 描述

迁移大量现有 UIProbeWindow 逻辑到 Service 时,必须保证用户可见行为零变化。本任务建立黄金样本回归机制:对模块录制输入->输出快照,迁移前后 diff 对比。这是后续所有 Service 抽离任务的安全网。

1. UIProbe/Tests/Editor/Golden/GoldenSampleRunner.cs
   - 提供 AssertGolden(name, actualOutput) 工具:首次运行写基线快照文件,后续运行与基线 diff,不一致则测试失败并输出差异。
   - 支持 string / CSV / 结构化 JSON 快照(对应未来 export_report 的 md/csv/json)。
   - 支持 --update-golden 等价开关(通过环境变量或常量)显式刷新基线,避免误覆盖。

2. UIProbe/Tests/Editor/GoldenSampleFixtures/
   - 放最小可控的输入夹具(若干 prefab 资源描述或内存假体预置数据),规模可控(避免大图 GetPixels 内存峰值)。
   - 录制至少一个示例基线(例如对一组预置 prefab 跑一个 fake 检测产出固定 Issue 列表快照)。

3. 与 Adapter 协作
   - Runner 通过内存假体喂入夹具数据,保证可重复、不依赖真实工程资源。

4. 不能动的边界
   - 本任务建立机制与示例基线,不为每个现有模块录制基线(那是各 Service 抽离任务的职责)。

## 验收标准

- [ ] GoldenSampleRunner 提供首次写基线、后续 diff 的能力
- [ ] 支持 string/CSV/JSON 三类快照
- [ ] 提供显式刷新基线的安全开关,避免误覆盖
- [ ] 至少录制一个示例黄金样本并能在 Test Runner 中通过
- [ ] 夹具数据规模可控,经内存假体喂入不依赖真实工程
- [ ] 故意改动输出会让黄金样本测试失败并打印差异
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)

## 退出准则

黄金样本机制可用,示例基线全绿,具备为后续 Service 迁移做行为零变化回归的能力。
