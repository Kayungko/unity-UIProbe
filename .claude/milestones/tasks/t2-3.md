## 描述

UICheckService 是其他 Unity MCP 难以提供的差异化能力,作为早期采用核心拉力。把综合检测、重名/重复检测(UIProbeWindow_DuplicateChecker.cs/_DuplicateCheckerBatch.cs)、过滤节点扫描(UIProbeWindow_FilterNodeScanner.cs)统一成结构化 Issue 模型。

1. UIProbe/Data/UIProbeChecker.cs
   - UICheckRequest(Targets:prefab 路径列表或全量、EnabledRules)。
   - UICheckReport(Issues:List<Issue>、Summary:CheckSummary 按 severity/rule 聚合、RanAt)。
   - 初始规则:重名节点、缺失 Sprite、缺失 Font、不必要 Raycast Target、空 Text、命名规范问题;重复检测与过滤节点扫描各作为一类规则。

2. UIProbe/Core/Services/UICheckService.cs(注入 PrefabIndexService)
   - RunChecks(request):基于 PrefabIndex 逐 prefab 跑规则产出结构化报告;全量大项目走 LongRunning+进度;无问题返回空 Issue 列表 + Summary。
   - GetCheckResults():读上次结果。
   - Issue 字段须完整(severity/ruleId/prefabPath/nodePath/componentType/message/suggestedFixId/canAutoFix),为 v0.2 preview/apply fix 预留。

3. Window 改造
   - DuplicateChecker/FilterNodeScanner 相关 UI 改用统一 Service 与 Issue 模型。

4. UIProbe/Tests/Editor/UICheckServiceTests.cs
   - 预置含已知问题的 prefab 集,验证各规则命中正确的 Issue。
   - 黄金样本对报告 JSON 做 diff 回归。

5. 不能动的边界
   - 只读不修改 prefab;Issue 模型统一,不为不同规则分裂多套结果结构。
