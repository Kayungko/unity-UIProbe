## 描述

UICheckService 是其他 Unity MCP 难以提供的差异化能力,作为早期采用核心拉力。把综合检测、重名/重复检测(UIProbeWindow_DuplicateChecker.cs/_DuplicateCheckerBatch.cs)、过滤节点扫描(UIProbeWindow_FilterNodeScanner.cs)统一成结构化 Issue 模型。

1. UIProbe/Data/UIProbeChecker.cs
   - UICheckRequest(Targets:prefab 路径列表或全量、EnabledRules)。
   - UICheckReport(Issues:List<Issue>、Summary:CheckSummary 按 severity/rule 聚合、RanAt)。
   - 初始规则:重名节点、缺失 Sprite、缺失 Font、不必要 Raycast Target、空 Text、命名规范问题;重复检测与过滤节点扫描各作为一类规则。

2. UIProbe/Core/Services/UICheckService.cs(注入 PrefabIndexService)
   - RunChecks(request):基于 PrefabIndex 逐 prefab 跑规则产出结构化报告;全量大项目走 LongRunning+进度;无问题返回空 Issue 列表 + Summary。
   - GetCheckResults():读上次结果。
   - Issue 字段须完整(severity/ruleId/prefabPath/nodePath/componentType/message/suggestedFixId/canAutoFix),为 v0.2 preview/apply fix 预留。

3. Window 改造
   - DuplicateChecker/FilterNodeScanner 相关 UI 改用统一 Service 与 Issue 模型。

4. UIProbe/Tests/Editor/UICheckServiceTests.cs
   - 预置含已知问题的 prefab 集,验证各规则命中正确的 Issue。
   - 黄金样本对报告 JSON 做 diff 回归。

5. 不能动的边界
   - 只读不修改 prefab;Issue 模型统一,不为不同规则分裂多套结果结构。

> **跨模块偏差说明**:7 条规则需逐节点的 UI 组件检测数据(Image/Text/Graphic),这些活在 Editor 加载的 prefab 树里(UnityEngine.UI API),
> Core.Services(全平台)够不到 → 补节点检视接缝 `IAssetGateway.InspectPrefab(prefabPath)` 返回中立 `IReadOnlyList<PrefabNodeRecord>`
> (DTO 定义在 Infrastructure 以避免 Infrastructure→Core.Services 循环,同 T2-2 的 `AssetReferenceRecord` 约定)。
> 生产适配层(Editor `UnityAssetGateway`)遍历 prefab 树产中立记录(`IsInteractable` 由适配层判定 = Selectable||ScrollRect,避免 Unity 类型层级泄漏进 Core);
> 规则在 Core.Services 跑于中立模型,逻辑层产契约 `Issue`(`UIProbe.Core.Contract`,单一来源),序列化为可读报告 DTO(Severity 转字符串,沿用 `UIDuplicateFakeFixture` 金样可读模式)。
> **原计划 `UIProbe/Data/UIProbeChecker.cs` 不动**:`UIProbe/Data/` 整体编译进 Editor-only 程序集(引用 UnityEditor),Core.Services 无法引用 → 新数据/报告类型落 `Core/Services/UICheckService.cs`(同 T2-1 的 `PrefabIndexData.cs` 约定)。
> **v0.1 范围裁剪**:① 省 `MissingCanvasGroupRule`(不在任务 6 类规则内,遗留默认关闭,无测试);② 报告省 `RanAt` 字段(Core.Services 无时钟接缝,YAGNI;M4 MCP 层可补盖时间戳)。
> **Window 改造(点 3)推后**:沿用 T2-1/T2-2 先例,本任务只交付 Service + 测试 + golden,DuplicateChecker/FilterNodeScanner UI 改接统一 Service 留待 M3/M4。
