# T1-1: 冻结 ToolContract 核心类型(代码落地)

## 元数据

- **Agent**: core-tool-contract
- **阶段**: prep
- **里程碑**: M1

## 描述

把 ToolContract.md 文档中冻结的契约类型落地为 C# 代码,作为后续所有 Service / Registry / Bridge / MCP 的单一来源。这是整个重构的接缝起点,必须先于任何 Service 抽离完成。

1. UIProbe/Core/Contract/ToolDescriptor.cs
   - 工具自描述模型:Name、Title、Summary、ParamsSchema(JSON Schema 字符串)、Profile(能力面标签)、ReloadSafe(bool)、LongRunning(bool)、UndoCapability(枚举 None/UnityUndo/FileBackup/MultiLevelStack)。

2. UIProbe/Core/Contract/ToolRequest.cs
   - 调用请求:ToolName、Params(原始 JSON)、RequestId、Mode(Preview/Execute 枚举)。

3. UIProbe/Core/Contract/ToolResult.cs
   - 统一结果:Status(枚举 Success/Cancelled/Interrupted/Failed)、Data(payload)、Changes(List<Change>)、Issues(List<Issue>)、Error(ToolError 可空)、JobId(可空,LongRunning 轮询用)。

4. UIProbe/Core/Contract/Change.cs
   - 单条变更描述:Kind、TargetPath、Before、After、UndoToken(可空)。

5. UIProbe/Core/Contract/Issue.cs
   - 结构化问题:Severity(枚举)、RuleId、PrefabPath、NodePath、ComponentType、Message、SuggestedFixId(可空)、CanAutoFix(bool)。

6. UIProbe/Core/Contract/ToolError.cs
   - Code(统一错误码字符串常量:UNITY_OFFLINE/UNITY_BUSY/MAIN_THREAD_TIMEOUT/DOMAIN_RELOAD_INTERRUPTED/TOOL_NOT_FOUND/INVALID_PARAMS/PERMISSION_DENIED/IO_ERROR 等)、Message、Retriable(bool)、Details(可空)。

7. UIProbe/Core/Contract/IUIProbeTool.cs 与 IUIProbeTool<TParams>
   - 工具接口:DescribeParams() 返回 schema、Validate(TParams) 返回 Issues、Preview(ctx, TParams) 返回 ToolResult、Execute(ctx, TParams) 返回 ToolResult。只读工具 Preview 可等价 Execute。

8. UIProbe/Core/Contract/ToolContext.cs
   - 执行上下文:CancellationToken、Progress 上报回调(节流)、注入的 Adapter 引用占位(IAssetGateway/IFileSystem/IEditorPrefs,接口在 T1-3 定义,这里先以 interface 引用)。

所有枚举与错误码常量必须与 ToolContract.md 文档逐字对齐,文档是单一来源,代码是其落地。本任务不实现任何具体工具,只产出契约骨架与编译通过。

## 验收标准

- [ ] ToolDescriptor/ToolRequest/ToolResult/Change/Issue/ToolError/ToolContext 全部以 C# 类型落地且字段与 ToolContract.md 一致
- [ ] ToolStatus 枚举含 Success/Cancelled/Interrupted/Failed
- [ ] UndoCapability 枚举含 None/UnityUndo/FileBackup/MultiLevelStack
- [ ] 统一错误码以常量形式集中定义,含 Retriable 语义
- [ ] IUIProbeTool<TParams> 含 DescribeParams/Validate/Preview/Execute 四阶段方法
- [ ] ToolContext 暴露 CancellationToken 与节流 Progress 回调
- [ ] Unity Editor 内编译通过,无具体工具实现
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)
