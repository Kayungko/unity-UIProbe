## 描述

把 MCP 的 ListTools/CallTool 翻译为 Bridge 的 /tools/list 与 /rpc,并以 serverId 为边界缓存工具描述。错误码透传不吞,保留 Retriable 语义。

1. mcp-server/src/tools/toolCache.ts
   - ToolCacheEntry(descriptor、fetchedAtServerId)。
   - serverId 变化即失效重拉(Domain Reload 后工具集可能变化)。

2. mcp-server/src/tools/toolProxy.ts
   - listTools():走缓存或 GET /tools/list 返回 McpTool[]。
   - callTool({name, arguments}):翻译为 POST /rpc 携带 token,返回 ToolResult;透传错误码(UNITY_OFFLINE/UNITY_BUSY/DOMAIN_RELOAD_INTERRUPTED/MAIN_THREAD_TIMEOUT/TOOL_NOT_FOUND)与 Retriable。

3. mcp-server/src/server.ts
   - 用 MCP SDK 注册 ListTools/CallTool handler,接 toolProxy。

4. mcp-server/test/toolProxy.test.ts
   - mock Bridge 验证:listTools 命中缓存;serverId 变化后重拉;callTool 翻译正确;错误码透传。

5. 不能动的边界
   - 一切经 Bridge /rpc,Server 不碰 Unity;错误不吞。
