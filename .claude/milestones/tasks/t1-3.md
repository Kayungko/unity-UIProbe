## 描述

现有业务大量直接调用 AssetDatabase / PrefabStageUtility / EditorPrefs / File 静态 API,Service 抽离后仍无法单测。本任务建立三个核心 Adapter 接口与其生产实现及测试假体,这是可测性的前提,没有它测试工程形同虚设。

1. UIProbe/Infrastructure/UnityAdapters/IAssetGateway.cs(放运行时程序集的接口部分,实现放 Editor)
   - FindAssets(filter) 返回 GUID 列表、LoadAssetAtPath、MoveAsset、GUIDToAssetPath、AssetPathToGUID。
   - 约定所有调用须在主线程(经 Dispatcher),接口文档注释标明。

2. UIProbe/Infrastructure/UnityAdapters/IFileSystem.cs
   - ReadAllText、WriteAllText、Exists、Backup(覆盖前备份,支持 FileBackup 撤销并返回 backup token)。
   - 写路径将来受 authorization 模块 write_allow/write_deny 约束(本任务只留接口,不接授权)。

3. UIProbe/Infrastructure/UnityAdapters/IEditorPrefs.cs
   - GetString/SetString/HasKey/DeleteKey。

4. UIProbe/Editor/Infrastructure/UnityAssetGateway.cs / UnityFileSystem.cs / UnityEditorPrefs.cs
   - 三个接口的生产实现,真实调用 AssetDatabase / File / EditorPrefs。

5. UIProbe/Tests/Editor/Fakes/InMemoryAssetGateway.cs / InMemoryFileSystem.cs / InMemoryEditorPrefs.cs
   - 内存假体:用 Dictionary 模拟资源表/文件表/prefs。假体需可控数据规模(为大图 GetPixels 类场景预留小规模可控数据)。

6. 不能动的边界
   - 本任务不改任何现有业务调用点,只新增接缝与实现/假体。把现有代码迁移到经接缝注入留到对应 Service 抽离的里程碑。

## 验收标准

- [ ] IAssetGateway/IFileSystem/IEditorPrefs 三接口定义完整,方法覆盖现有主要静态依赖
- [ ] 三个 Unity 生产实现真实调用对应静态 API 且编译通过
- [ ] 三个内存假体可独立工作,不依赖 Unity 运行环境
- [ ] IFileSystem.Backup 返回可用于撤销的 backup token(对应 UndoCapability.FileBackup)
- [ ] 接口注释标明 IAssetGateway 调用须在主线程
- [ ] 假体支持可控数据规模设置
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)

## 退出准则

三组 Adapter 接口/实现/假体就位,可被 Service 构造注入,内存假体可在无 Unity 运行环境下单测。
