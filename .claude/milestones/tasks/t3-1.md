# T3-1: MainThreadDispatcher:后台线程到主线程的执行边界

> 由 milestone_gen 生成。

## 元数据

- **Agent**: core-unity-bridge
- **阶段**: exec
- **里程碑**: M3

## 描述

HTTP 监听在后台线程,Unity API 必须在主线程执行,需要统一投递边界。本任务实现 MainThreadDispatcher,经 EditorApplication.update 逐帧 drain 并发队列、主线程执行、结果写回 TaskCompletionSource。这是 Bridge 正确性的核心。

1. UIProbe/Editor/Infrastructure/Bridge/MainThreadDispatcher.cs
   - DispatchJob(JobId、Action/TCS、Status:Running/Done/Interrupted)。
   - Enqueue(Func) 返回 Task:后台线程入并发队列(ConcurrentQueue),await TCS,默认 30s 超时 -> MAIN_THREAD_TIMEOUT 并尽量取消。
   - EditorApplication.update 回调中 drain 队列、主线程执行、写回 TCS。
   - isCompiling/isUpdating 时暂缓 drain 写操作(只读可放行),放行写操作前等 isCompiling=false 稳定窗口。
   - 执行单元短小、可在 Stage 边界检查取消。

2. 长任务支持
   - LongRunning 工具立即返回 jobId,主线程后台分帧执行,结果存 job 表;进度经 ToolContext.Progress 节流上报。

3. UIProbe/Tests/Editor/MainThreadDispatcherTests.cs
   - 验证:入队任务最终在主线程执行并返回;超时回 MAIN_THREAD_TIMEOUT;isCompiling 时写操作暂缓。

4. 不能动的边界
   - 任何 Editor API 调用必须经此 Dispatcher;Service 层不在监听线程直接碰 Unity API。

## 验收标准

- [ ] Dispatcher 经 EditorApplication.update 逐帧 drain 并发队列并在主线程执行
- [ ] Enqueue 返回可 await 的 Task,默认 30s 超时回 MAIN_THREAD_TIMEOUT 并尝试取消
- [ ] isCompiling/isUpdating 时暂缓写操作 drain,放行前等稳定窗口
- [ ] LongRunning 立即返回 jobId,进度经 ToolContext.Progress 节流上报
- [ ] 执行单元可在 Stage 边界检查取消
- [ ] 单测覆盖主线程执行/超时/编译中暂缓
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)

## 退出准则

MainThreadDispatcher 可把后台请求安全投递到主线程执行,超时与编译中暂缓行为正确,长任务 jobId 机制可用。
