## 描述

统一处理某资源被哪些 prefab/节点/组件使用。基于 PrefabIndex 派生不另存副本,保证引用结果与索引一致。从 UIProbeWindow_AssetReferences.cs 抽出查询与导出逻辑。

1. UIProbe/Core/Services/AssetReferenceService.cs(注入 PrefabIndexService + IFileSystem)
   - AssetReferenceQuery(AssetPath/AssetName/Guid/SpriteName 可选 + ReferenceTypeFilter,至少一个维度)。
   - AssetReferenceResult(AssetPath/PrefabPath/NodePath/ComponentType/ReferenceType)。
   - FindReferences(query):基于 PrefabIndex 派生引用列表;大结果集走流式/分页避免一次性物化;资源未被引用返回空列表(非错误);PrefabIndex 未构建时提示先 build。
   - ExportCsv(results, options):写受控目录返回 reportPath。

2. UIProbeWindow_AssetReferences.cs 改造
   - UI 保留,数据改由 Service 提供。

3. UIProbe/Tests/Editor/AssetReferenceServiceTests.cs
   - 用预置 PrefabIndex(经内存假体)验证按路径/GUID/Sprite 名/引用类型查询。
   - 黄金样本对一组查询的 CSV 导出做 diff 回归。

4. 不能动的边界
   - 只读,不自建索引,严格从 PrefabIndex 派生。

## 验收标准

- [ ] AssetReferenceService 提供 FindReferences/ExportCsv
- [ ] 支持按 AssetPath/AssetName/Guid/SpriteName/ReferenceType 过滤,至少一维度否则 INVALID_PARAMS
- [ ] 结果严格从 PrefabIndex 派生,不另存副本
- [ ] 资源未被引用返回空列表而非错误
- [ ] PrefabIndex 未构建时给出先 build 的提示
- [ ] 大结果集走流式/分页
- [ ] ExportCsv 写受控目录返回 reportPath
- [ ] UI 行为不变,黄金样本 CSV 导出 diff 全绿
- [ ] Includes automated geometric assertions (e.g., Vector3 position within ±0.1, quaternion |q|=1.0 ±1e-6, mesh bounds non-degenerate)
